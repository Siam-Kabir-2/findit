FindIt
Motivation

The campus problem

Lost phones, IDs, bags, and notebooks pile up in offices with no shared digital record — owners struggle to search, claim, and prove ownership.

Scattered notices

Paper boards and chat groups are hard to search and easy to miss.

No verification

Anyone can claim an item without proof or admin review.

No audit trail

Status changes are not logged — disputes are hard to resolve.

FindIt
Objectives

What FindIt delivers

For students
  • Register & log in securely
  • Browse / filter lost & found board
  • Report items with photos
  • Submit claims with proof text
  • Track claim status on a dashboard
For admins
  • Separate admin console & auth
  • Approve / reject claims via PL/SQL
  • Manage items, users, categories, locations
  • Monitor stats from package functions
  • Inspect Oracle audit logs

Architecture

System architecture

flowchart LR U[Student Browser] --> L[Laravel Blade App] A[Admin Browser] --> L L --> C[Controllers] C --> M[Eloquent Models] C --> S[FinditPlsqlService] M --> O[(Oracle 11g XE)] S --> P[findit_pkg] P --> O T[DB Triggers] --> AL[(audit_logs)] O --> T

Reads often go through Eloquent; critical writes (register, add item, claim approve/reject) go through the PL/SQL package.
